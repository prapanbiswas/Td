🏗️ Architecture Insight: Why This Tool Runs Locally

      
The shade generation algorithm converts the seed hex color to HSL (Hue-Saturation-Lightness) color space using pure JavaScript math, then applies a perceptually-uniform lightness curve across the 11 standard Tailwind shade stops (50–950). Saturation is slightly boosted at mid-range shades (400–600) to match Tailwind's visual design language. The entire color computation — hex→HSL→shade generation→HSL→hex — runs as arithmetic in your browser. The resulting config strings are assembled using template literals. No color computation library is needed.

    

    

      
❓ Frequently Asked Questions

      

        
How do I add a custom Tailwind color palette shade to my tailwind.config.js?
Copy the "tailwind.config.js" export from this tool and paste the color object into your Tailwind config's theme.extend.colors (to add alongside defaults) or theme.colors (to override defaults). For example: module.exports = { theme: { extend: { colors: { brand: { 50: '#f5f3ff', ... 950: '#2e1065' } } } } }

        
Why do the generated shades look different from Tailwind's built-in palettes?
Tailwind's built-in palettes are hand-curated by designers for optimal perceptual uniformity and accessibility contrast ratios. The algorithmic generation here uses an HSL-based lightness curve which produces good results but may not perfectly replicate the manual curation. For production design systems, treat the generated palette as a starting point and adjust individual shade values as needed for your brand identity.

        
Can I use the exported CSS variables directly in a Tailwind CSS v4 project?
Yes. Tailwind CSS v4 uses CSS custom properties natively. Copy the "CSS Variables" export and add it to your @theme block or :root section. The variable naming convention uses --color-{name}-{shade} which aligns with Tailwind v4's theme variable convention.

        
How do I check if my generated palette has sufficient contrast for WCAG accessibility?
As a general rule, text on a shade-900 background should use shade-50 or white, and text on a shade-100 background should use shade-900 or black to meet WCAG AA contrast requirements (minimum 4.5:1 for normal text). Use a dedicated contrast checker tool after generating your palette to verify specific shade combinations. The generated shade-500 color is typically closest to your seed color and works well as a primary interactive color with white text on dark backgrounds.
