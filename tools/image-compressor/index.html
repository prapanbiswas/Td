User Manual

  1. Drop or select a JPEG, PNG or WebP image.
  2. Choose the output format (JPEG is smallest for photos).
  3. Drag the quality slider — 75–85% is the sweet spot for most images.
  4. Click Compress Image to see the side-by-side preview and savings.
  5. Click Download to save the compressed file.

Architecture Insight

The compressor draws your image onto an HTML5 Canvas element, then calls canvas.toBlob()with your chosen MIME type and quality value (0–1). This leverages the browser's native codec — the same engine used by the OS — with zero server involvement. PNG compression is lossless so the quality slider controls filtering only.

FAQ

Does my image get uploaded anywhere?
No. The entire operation happens inside your browser using the Canvas API. Nothing is sent to any server.
What quality setting should I use?
For photos: 75–85% JPEG gives excellent quality at roughly 70–80% size reduction. For graphics with text: use PNG or WebP at 90%+.
Why is my PNG not smaller after compression?
PNG is lossless — re-exporting as PNG may not reduce size. Switch the format to JPEG or WebP for much better compression on photographic images.
Is WebP supported by all browsers?
WebP is supported in all modern browsers (Chrome, Firefox, Safari 14+, Edge). It's an excellent choice for web images.
