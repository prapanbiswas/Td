⚙️ Architecture Insight

The optimizer uses regex and DOM parsing entirely in JavaScript. It strips XML comments, collapses redundant whitespace between tags, removes Adobe/Inkscape editor namespaces, and deletes non-rendering elements like <metadata>, <title>, and <desc>. No libraries or WASM are used.

❓ FAQ

Will this break my SVG's appearance?
The default options only remove non-visual content (comments, metadata, editor data). The shapes, paths, and styles are never modified.
How much smaller will my SVG be?
SVGs exported from Illustrator or Inkscape often contain 30–60% bloat from editor metadata and whitespace. Expect 20–50% size reduction.
Should I remove <title> tags?
For decorative icons, yes — titles add weight and aren't displayed. For SVGs used as content (e.g. charts), keep titles for accessibility.
