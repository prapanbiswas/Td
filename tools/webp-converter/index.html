Architecture Insight

Images are decoded natively by the browser, drawn to a Canvas, and re-encoded to image/webp using toBlob(). GIF animations are flattened to a single WebP frame (the first frame). No external libraries are used.

FAQ

Why convert to WebP?
WebP provides 25–34% better compression than JPEG at equivalent visual quality, resulting in faster page loads and lower bandwidth costs.
Will GIF animations be preserved?
No — GIFs are flattened to the first frame. For animated WebP, you'd need a dedicated animation tool.
Does transparency survive conversion?
Yes. PNG transparency is preserved when converting to WebP. JPEG files have no transparency channel so none is added.
