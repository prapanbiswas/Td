📖 User Manual & Operational Guide

  1. Enter your pattern: Type a JavaScript-compatible regex pattern in the input field (without surrounding slashes). Use the Quick Insert buttons for common patterns.
  2. Toggle flags: The g (global), i (case-insensitive), m (multiline), and s (dotall) flag buttons are active when highlighted purple. Toggle them as needed.
  3. Enter test text: Paste or type your test string in the "Test String" area. The highlighted result shows matched text in amber.
  4. Review matches: The Matches panel lists every match with its index position, matched value, and any capture group contents color-coded by group number.

🏗️ Architecture Insight: Why This Tool Runs Locally

This tool uses JavaScript's native RegExp constructor and String.prototype.matchAll() to execute your regex pattern against the test string entirely in your browser's JavaScript engine. Match positions are used to split the test string into text and highlight spans, which are injected into the DOM as a series of <span> elements. No regex is sent to a server. No test string leaves your device. The entire evaluation pipeline runs at native browser speed.

❓ Frequently Asked Questions

What regex flavor does this visual regular expression compiler use?
This tool uses JavaScript's built-in ECMAScript RegExp engine, which is implemented in every modern browser's V8, SpiderMonkey, or JavaScriptCore engine. It supports standard features including character classes ([a-z]), anchors (^, $), quantifiers (*, +, ?, {n,m}), groups ((), (?:), (?<name>)), lookaheads, and Unicode escapes. PCRE-specific features like (?<=) lookbehinds are supported in modern browsers but not in older engines.
How do I test a regex that spans multiple lines?
Enable the m (multiline) flag to make ^ and $ match the start and end of each line rather than the entire string. Enable the s (dotall) flag to make the dot . match newline characters as well. Both flags can be active simultaneously. Your test string can include newlines — press Shift+Enter in the textarea to insert a newline.
Can I use named capture groups in the regex pattern?
Yes. JavaScript's regex engine supports named capture groups using the syntax (?<name>...). Named groups appear in the match results alongside numbered groups. For example, the pattern (?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2}) extracts date components with clear semantic names.
Why does my regex work in Python or PHP but not in this live graph compiler?
Different regex engines have different syntax rules. Python uses PCRE-compatible syntax, PHP uses PCRE, while JavaScript has its own ECMAScript syntax. Common differences: Python uses \A and \Z for string anchors (use ^ and $ in JS), Python supports recursive patterns (not in JS), and some POSIX character classes differ. The Quick Insert buttons provide JS-compatible equivalents for common patterns.
