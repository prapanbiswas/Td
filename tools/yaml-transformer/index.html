🏗️ Architecture Insight: Why This Tool Runs Locally

      
This tool implements a JavaScript YAML parser that converts YAML to a JavaScript object graph, then serializes to JSON using JSON.stringify(). The YAML parser handles the most common YAML syntax: scalars (strings, numbers, booleans, null), block sequences, block mappings, inline arrays and objects, multiline strings, and quoted strings. The JSON-to-YAML direction uses a recursive serializer that produces indented YAML block format. Both conversions run synchronously in browser memory with no network I/O.

    

    

      
❓ Frequently Asked Questions

      

        
How do I convert YAML frontmatter to a JSON object instantly offline?
Select the "Extract Frontmatter" mode and paste your Markdown file content (including the --- delimiters). The tool extracts the YAML block between the opening and closing --- markers and converts it to a JSON object. This is useful for processing Jekyll, Hugo, Gatsby, or Astro static site content files.

        
What YAML features are supported?
The built-in parser supports standard YAML frontmatter syntax used in static site generators and configuration files: scalar strings (quoted and unquoted), integers, floats, booleans (true/false/yes/no), null values, block sequences (list items prefixed with -), and nested block mappings (key: value indented). Advanced YAML features like anchors, aliases, multi-document streams, and custom tags are not supported by the lightweight parser.

        
Can I use this to convert Kubernetes or Docker Compose YAML to JSON?
Yes, for standard Kubernetes and Docker Compose YAML files. Most production k8s manifests and compose files use standard YAML features (scalars, sequences, mappings) that are fully supported. Multi-document YAML files (multiple documents separated by ---) are parsed as the first document only.

        
Why does my YAML convert to JSON with incorrect types?
YAML automatically infers scalar types. Values like true, yes, on, false, no, off are treated as booleans. Numeric strings like 1e3 are treated as numbers. To force a value to remain a string, quote it: version: "1" instead of version: 1. This is standard YAML 1.2 behavior.
